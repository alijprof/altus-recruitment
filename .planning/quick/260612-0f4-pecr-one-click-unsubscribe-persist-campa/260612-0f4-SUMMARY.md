# Phase quick-260612-0f4: PECR One-Click Unsubscribe + Personalisation Persistence

**One-liner:** PECR-compliant one-click unsubscribe with real List-Unsubscribe-Post headers (node:crypto token, RFC 8058 GET/POST route, durable suppression column) plus IN-02 personalised_intro/outro persistence on campaign send.

## What Was Built

### Task 1 (commit 912db08) — Migration + middleware allowlist

**Migration `20260612000000_unsubscribe_tokens_and_suppression.sql`** adds:
- `email_campaign_recipients.unsubscribe_token text` — nullable, with a partial unique index (`email_campaign_recipients_unsub_token_idx`) for O(log n) token lookup. NULL for legacy rows.
- `candidates.email_marketing_unsubscribed_at timestamptz` — the durable PECR consent-withdrawal flag (NULL = consented). Column comment documents both getCampaignSegment and send-loop gates.

**`src/lib/supabase/middleware.ts`** — `/unsubscribe` added to `PUBLIC_PATHS` with rationale comment. The existing `pathname.startsWith()` branch matches `/unsubscribe/{token}` automatically.

### Task 2 (commits a6c23b5 RED, 1296658 GREEN) — Unsubscribe helpers + unit tests

**`src/lib/email/unsubscribe.ts`** (server-only):
- `generateUnsubscribeToken()` — `node:crypto randomBytes(32)` base64url, ~256 bits entropy. No UUID, no Math.random.
- `buildUnsubscribeUrl(token, baseUrl)` — single source-of-truth for the URL. `encodeURIComponent` on the path segment (T-0f4-REDIRECT). Falls back to `altusrecruit.com` when `NEXT_PUBLIC_SITE_URL` is unset.
- `maskEmail(email)` — first+last local char, rest asterisks, domain intact. Never throws on edge cases.
- `suppressByToken(supabase, token)` — indexed token lookup, re-reads before write (idempotent), org-scoped candidate update (T-0f4-XTENANT), returns `{ ok, alreadyUnsubscribed }` discriminated union, never throws, no PII in Sentry tags.

**`src/lib/email/unsubscribe.test.ts`** — 19 unit tests: token entropy/charset/uniqueness, URL join/fallback/encoding, maskEmail edge cases, suppressByToken stub branches (lookup, idempotency, error path).

### Task 3 (commit 31cfbb4) — Route + segment gate + headers + persistence

**`src/app/(public)/unsubscribe/[token]/route.ts`** — Single file, no page.tsx (would be a build error). Inline HTML only, no React components.
- `GET`: safe idempotent read. Service-role token lookup. If found → confirm page with `maskEmail` and form POST to same URL. If not found → constant generic copy, status 200 (no oracle, T-0f4-ENUM). Comment explains GET must have no side effects (T-0f4-PREFETCH).
- `POST`: handles both form submit and RFC 8058 one-click (no body validation). `suppressByToken` awaited before 2xx (RFC 8058 §3). Returns identical constant confirmation HTML regardless of outcome.

**`src/lib/db/campaigns.ts`**:
- `getCampaignSegment` — added `.is('email_marketing_unsubscribed_at', null)` belt gate (escape hatch for pre-regen types).
- `insertCampaignRecipients` — generates `unsubscribe_token` per recipient at insert time via `generateUnsubscribeToken()`.
- `updateRecipientStatus` — extended with `personalisedIntro`/`personalisedOutro` options; uses `as unknown as` escape hatch on the update patch.

**`src/lib/inngest/functions/send-email-campaign.ts`**:
- **PRE-LAUNCH BLOCKER (WR-09) resolved**: `mailto:` placeholder replaced with real `buildUnsubscribeUrl(token, env.NEXT_PUBLIC_SITE_URL)`.
- `List-Unsubscribe-Post: List-Unsubscribe=One-Click` header added.
- `List-Unsubscribe` header URL is byte-for-byte equal to the POST endpoint URL (RFC 8058).
- Send-time suppression re-check (braces): if `email_marketing_unsubscribed_at` is non-null, mark `suppressed_unsubscribed` and skip.
- Legacy-token persist: generates token, persists it (error-checked), marks `token_persist_error` and skips if persist fails (T-0f4-TOKENGAP).
- IN-02 closure: `personalisedIntro` + `personalisedOutro` written on the `sent` path.
