## Phase Details

### Phase 1: Internal ATS

**Goal:** Anchor customer can use the app internally to manage candidates, clients, jobs, and a basic pipeline — with AI-powered CV parsing and GDPR compliance baked in.
**Mode:** mvp
**Depends on:** Nothing (brownfield — Tasks 1–2 already merged)
**Requirements:** FOUND-01, FOUND-02, FOUND-03, FOUND-04, FOUND-05, FOUND-06 (Complete), CAND-01, CAND-02, CAND-03, CAND-04, CAND-05, CAND-06, CAND-07, CV-01, CV-02, CV-03, CV-04, CV-05, CLIENT-01, CLIENT-02, CLIENT-03, CLIENT-04, CLIENT-05, PIPE-01, PIPE-02, PIPE-03, PIPE-04, PIPE-05, PIPE-06, DASH-01, DASH-02, DASH-03, DASH-04, DASH-05, DASH-06

**Success Criteria** (what must be TRUE when this phase completes):
1. Recruiter can create a candidate with GDPR consent captured, view and edit all fields, log calls and notes from the detail page — and be blocked from creating without consent
2. Recruiter can upload a CV, wait for AI parsing to complete in the background, and review/accept extracted structured data (name, role, skills, salary, seniority) from a review panel
3. Recruiter can create a client, add contacts nested under it, and see a combined activity timeline for the client across contacts and jobs
4. Recruiter can create a job against a client, add candidates as applications, drag cards between pipeline stages on a kanban board, and decline a candidate with a structured reason — all stage changes auto-logged to activities
5. Authenticated home shows real organisation metrics (candidate count, active jobs, open applications), a recent activity feed, stale-application alerts, and candidates needing follow-up
6. Recruiter can invite a teammate by email; teammate signs up and joins the same organisation

### Phase 2: Search, Match & Intake

**Goal:** Recruiters can find the right candidate for a job in seconds using natural language, candidates can apply publicly, and inbound Outlook (Microsoft 365) activity is captured automatically.
**Mode:** mvp
**Depends on:** Phase 1
**Requirements:** SEARCH-01, SEARCH-02, SEARCH-03, SEARCH-04, MATCH-01, MATCH-02, MATCH-03, APPLY-01, APPLY-02, EMAIL-01

**Success Criteria** (what must be TRUE when this phase completes):
1. Recruiter can type "senior Python engineer with offshore wind experience in Aberdeen" into search and receive ranked candidates ordered by semantic similarity — no keyword matching required
2. Recruiter can open a job and see the top-matching candidates surfaced automatically, each with a 0–100 score, 2–3 strengths, 1–2 gaps, and suggested screening questions generated by Sonnet
3. A candidate can navigate to the public apply form, upload their CV, give GDPR consent, and appear as a new candidate record with CV parsing triggered automatically
4. Recruiter can connect Outlook (Microsoft 365) via OAuth and see inbound emails to/from candidates and contacts logged to activity timelines automatically — no manual copy-paste

### Phase 3: LinkedIn Capture, Spec Workflow & Shortlists

**Goal:** Recruiters can capture LinkedIn profiles in one click, structure a job from a recorded spec call, generate inclusive job ads, and maintain per-job shortlists — making the tool competitive with Firefish core for perm recruitment.
**Mode:** mvp
**Depends on:** Phase 2
**Requirements:** LINKEDIN-01, SPEC-01, SPEC-02, AD-01, SHORT-01, SHORT-02, REPEAT-01, REPEAT-02

**Success Criteria** (what must be TRUE when this phase completes):
1. Recruiter can open a LinkedIn profile in Chrome, click the extension, and have a new (or updated) candidate record appear in the CRM with embedding populated — without any form filling
2. Recruiter can upload a spec call recording; Whisper transcribes it; Sonnet produces a structured draft JD (title, location, salary range, must-haves, nice-to-haves, culture, urgency); recruiter reviews and approves before the job is created
3. Recruiter can generate a job ad from an approved JD with one click, and receive an inclusivity score with specific improvement suggestions on any existing ad
4. Recruiter can build a per-job shortlist (working set separate from formal applications), float a candidate speculatively without a specific job, and see which clients have gone silent with a one-click outreach hook
5. Source attribution report shows placements per source so the recruiter can see which channels produce the best ROI

**Plans:** 7/7 plans complete

- [x] 03-02-spec-audio-jd-PLAN.md — spec audio upload → Whisper → Sonnet JD draft → review/approve → jobs row (SPEC-01, SPEC-02)
- [x] 03-03-shortlists-floats-PLAN.md — `shortlist` enum + nullable `job_id` + per-job shortlist tab + candidate floats (SHORT-01, SHORT-02)

### Phase 4: Voice, Marketing & Reporting

**Goal:** Recruiters can dictate voice notes that update the CRM, run personalised email campaigns, and answer natural-language questions about their desk's performance — giving the anchor customer board-ready reporting.
**Mode:** mvp
**Depends on:** Phase 3
**Requirements:** MARKET-01, MARKET-02, MARKET-03, REMIND-01, VOICE-01, VOICE-02, REPORT-01, REPORT-02

**Success Criteria** (what must be TRUE when this phase completes):
1. Recruiter can dictate a 30-second voice note after a meeting; Sonnet extracts key points, a stage update recommendation, and action items; recruiter approves any field changes before they apply — no data changes without approval
2. Recruiter can build a segmented email campaign (e.g. "actively looking candidates in Aberdeen") and send personalised emails via Resend, each tailored to the recipient's CV and recent activity — after explicit approval step, no auto-send
3. Automated reminders surface in the dashboard for stale candidates (30+ days no contact, ordered by market status urgency) and dormant clients — recruiter acts from the notification without context-switching
4. Recruiter can type "how many placements did we make last quarter by sector?" and receive a correct answer from a read-only SQL query validated against an allowlist — no hallucinated data
5. Buyer-value dashboards show placements per recruiter per quarter, time-to-fill by sector, source ROI, pipeline value, and commission summary — ready for an acquirer's due-diligence view

### Phase 5: SaaS Shell

**Goal:** The product is ready for a second paying customer: self-service signup, Stripe billing, per-org branding, super-admin tooling, and a public documentation site.
**Mode:** mvp
**Depends on:** Phase 4
**Requirements:** SAAS-01, BILL-01, BRAND-01, ADMIN-01, MARKETING-01

**Success Criteria** (what must be TRUE when this phase completes):
1. A new agency can sign up without any manual intervention — create an org, complete an onboarding tour, import a CSV of candidates, and be using the product within 30 minutes
2. The new org is on a Stripe subscription; the billing portal lets them upgrade, downgrade, or cancel; the CRM enforces plan limits
3. Each org can upload their logo and set brand colours which appear on their public careers/apply site
4. Super-admin can impersonate any org, override plan limits, and review per-tenant AI usage costs from a support console
5. The public marketing site explains the product, links to documentation, and has a status page — ready to share with prospects
