### Task 0.2 — ffmpeg availability probe + Sentry Phase-3 tag conventions

**Type:** code (auto, tdd="true")

- NEW `src/lib/inngest/functions/probe-ffmpeg.ts` — one-shot Inngest function triggered by `ops/probe-ffmpeg` event; runs `ffmpeg -version` and logs to Sentry with `level: 'info'`, tag `layer: 'inngest', function: 'probe-ffmpeg'`. Pattern: `src/lib/inngest/functions/refresh-outlook-subscription.ts` heartbeat shape (lines 49–55 per PATTERNS §2)
- EDIT `src/app/api/inngest/route.ts` — register `probeFfmpeg` in `functions: [...]` array (alphabetical insertion per PATTERNS §2)
- NEW `docs/phase-3-sentry-tags.md` — convention table (one-pager): every Phase 3 captureException MUST include `phase: 'p3'` plus `layer` ∈ {`route-handler`, `action`, `db`, `inngest`, `ai-wrapper`, `extension-ingest`} and `function|helper|route` slug. Reference PATTERNS §10.

**Detail:**
- `ffmpeg.ts` exports BOTH:
  - `recompressToOpus(input: Buffer, opts: { bitrate: string; channels: number }): Promise<Buffer>` — recompress to mono 32 kbps Opus
  - `probeDurationSeconds(input: Buffer): Promise<number>` — wraps `fluent-ffmpeg().ffprobe()` against a `PassThrough` source, returns `format.duration` rounded to nearest integer. **Required by Plan B Task B.2** to derive the input-token estimate for `ai_usage.record_ai_usage(p_input_tokens, ...)` on every Whisper call. CRITICAL-2 fix (plan-check 2026-05-19) — without this, Plan B's Inngest body has an unresolved placeholder and either fails compile or writes wrong cost data.
- Mirror voyage.ts: `import 'server-only'`, `import 'fluent-ffmpeg'`, singleton `ffmpegPath = require('@ffmpeg-installer/ffmpeg').path` set at module load, `import.meta.url`-safe. Use `fluent-ffmpeg().setFfmpegPath(ffmpegPath)`. Pipe through a `PassThrough` stream; resolve with the buffered output. On error: `Sentry.captureException(new Error(\`ffmpeg: ${err.name}\`))` per the "VERIFICATION R4" rule (PATTERNS §1 — never the raw error).
- `probe-ffmpeg.ts` Inngest function: trigger `{ event: 'ops/probe-ffmpeg' }`, single `step.run('probe', async () => { const { stdout } = await execFile(ffmpegPath, ['-version']); return stdout.split('\n')[0]; })`. Sentry breadcrumb on success: `Sentry.captureMessage('phase3:ffmpeg:probe:ok', { level: 'info', tags: { phase: 'p3', layer: 'inngest', function: 'probe-ffmpeg' } })`. RESEARCH §Environment Availability — "**Wave-0 verification: run `ffmpeg -version` from a deployed Inngest function once and confirm.**"
- TDD: Vitest test `ffmpeg.test.ts` mocks `fluent-ffmpeg` and asserts:
  (a) `recompressToOpus()` passes `-c:a libopus -b:a 32k -ac 1` flags (per RESEARCH §"Don't Hand-Roll", Pattern 3 reference).
  (b) `probeDurationSeconds(<fixture buffer>)` returns the expected integer seconds for a known fixture file (CRITICAL-2 test).
- `phase-3-sentry-tags.md`: 1-page table mapping every new file in Phase 3 to its required Sentry tag set. Use as a checklist by every downstream plan.

**Acceptance:**
- `pnpm test -- --run src/lib/ai/ffmpeg.test.ts` passes (recompress flags asserted).
- `grep -c "phase: 'p3'" docs/phase-3-sentry-tags.md` >= 1.
- After Wave 0 deploy, manually fire `inngest.send({ name: 'ops/probe-ffmpeg', data: {} })` once via the Inngest dashboard and verify a `phase3:ffmpeg:probe:ok` breadcrumb appears in Sentry.
